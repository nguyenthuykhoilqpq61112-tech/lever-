# 证据核验与主动检索

> 使用边界：当涉及当年政策、院校状态、招生计划、位次、专业组、学费、章程、投档线、征集志愿、合并更名或趋势判断时读取本文档；产出信源优先级、当年资料刷新清单、核验清单和证据标签；不要用非官方线索直接支撑确定结论。

当涉及当年政策、院校状态、招生计划、录取位次、专业组、学费、章程、合并更名、转专业政策等可能变化的信息时，必须读取本文档。

## 主动检索规则

以下场景必须主动检索或要求用户提供官方资料：

- 用户要求具体到某一年、某省份的志愿填报。
- 用户要求预测录取概率、调档线或位次。
- 用户询问某校是否合并、更名、升格、转设、停招、换校区。
- 用户询问某专业组包含哪些专业、是否服从调剂、是否有限制条件。
- 用户询问学费、中外合作、住宿校区、身体要求、单科要求、语种要求。
- 用户询问保研率、推免资格、推免去向、考研率、考研通过率、升学率、考研去向或“考研神校”。
- 用户要求最终志愿排序表。
- 用户要求全量候选池、全国范围搜索、大规模筛选或多轮候选收敛。
- 用户要求判断 AI、产业周期、人口老龄化、区域国别、一带一路、绿色低碳、供应链、公共治理等长期趋势。

若无法联网或无法获取当年资料，必须说明“当前只能做模拟方案”，并列出待核验事项。

检索前先列出已知事实、关键未知项和验证假设；检索后必须说明资料如何改变候选池、风险等级或排序。

## 当年实时数据刷新机制

当任务处于出分后、志愿填报期、投档录取期、征集志愿期，或用户要求最终志愿表时，先执行一次“当年资料刷新”。目的不是保证所有资料都已经发布，而是判断哪些当年官方资料已经出现、哪些尚未发布、哪些发生更新，并用更高优先级的当年资料替换历史资料。

### 刷新优先级

1. 当年官方已发布：省级考试院政策文件、省控线、一分一段、招生计划、计划变更、志愿填报说明、院校专业组、招生章程、分省分专业计划、投档线、征集志愿公告和录取结果统计。
2. 当年官方尚未发布：标注为“当年官方未发布”，不得用近年数据伪装成今年结论。
3. 近年官方资料：可用于模拟、波动区间和待核验判断，必须注明年份和口径。
4. 第三方工具、商业数据、社交媒体或家长群：只能作为线索，必须回到官方资料交叉核验。

### 按阶段刷新

| 阶段 | 优先刷新 | 对建议的影响 |
| --- | --- | --- |
| 出分前 | 招生工作规定、招生章程、招生计划、选科要求、特殊类型规则 | 只能做规则预案和候选雷达，不做正式位次判断 |
| 出分后 | 省控线、一分一段、批次资格、同分排序 | 位次优先替换分数判断，重新划分冲稳保范围 |
| 填报期 | 官方计划查询、院校专业组、计划变更、学费、校区、调剂范围 | 更新候选池、专业组风险和最终排序 |
| 投档期 | 本批次投档线、院校专业组投档分、投档最低位次 | 校准前期判断，识别明显重新定价的学校或专业组 |
| 征集期 | 征集志愿计划、缺额专业、补录条件、时间窗口 | 仅用于征集/补录策略，不反推前期正式批次必然性 |
| 录取后 | 专业录取分、录取统计、实际去向 | 用于复盘、下一年模型和入学后规划，不作为当年未结束批次的保证 |

### 刷新动作

- 先确认省份、年份、批次、科类/选科和当前时间节点。
- 优先检索省级考试院、官方志愿填报系统、阳光高考/阳光志愿和高校本科招生网。
- 对每个资料记录发布时间、更新日期、适用省份/批次/科类、附件或页面名称、来源链接或资料名称。
- 如果找到当年官方新资料，更新证据标签、候选池 ledger、风险等级和待核验事项。
- 如果官方资料未发布，明确写“当年官方未发布”，用近年资料只做模拟区间。
- 如果同一事项存在冲突，以当年最新官方资料为准；仍无法判断时，下调置信度并列入待核验 ledger。
- 如果第三方平台声称已有今年分数线、投档线或预测概率，必须核对其是否引用官方公告；无法核对时只作为线索。

## 信源优先级

优先级从高到低：

1. 省级教育考试院、招生考试院、官方志愿填报系统。
2. 官方招生计划书、一分一段表、批次控制线、投档线和征集志愿公告。
3. 高校本科招生网、招生章程、专业组说明、招生计划和历年录取数据。
4. 教育部、省教育厅、主管部委、事业单位登记和高校设置公告。
5. 高校就业质量报告、本科教学质量报告、学科建设、专业认证、硕博点公告。
6. 可靠数据平台和志愿填报工具，仅作辅助交叉验证。
7. 社交媒体、论坛、家长群、短视频和口碑信息，只能作为线索，不可单独作为结论。

## 外部数据源分级

外部数据源必须记录覆盖范围、更新时间、许可证/付费或调用限制、字段口径和交叉核验状态。不要把“可调用”写成“权威可靠”。

| 等级 | 典型来源 | 可用于 | 限制 |
| --- | --- | --- | --- |
| 官方权威 | 省级考试院、官方志愿系统、阳光高考、阳光志愿、高校招生网、招生章程 | 当年规则、招生计划、专业组、章程和正式公告 | 分散、格式不一，未必有统一开放 API |
| 官方可查询/下载 | 一分一段表、省控线、投档线、征集志愿公告、高校历年录取数据 | 位次模拟、历史参考和规则核验 | 跨省跨年口径差异大，需要版本化 |
| 商业 API/数据集 | 咕咕数据、CnOpenData、第三方志愿工具导出 | 辅助补全历史录取、专业分数和候选发现 | 需付费或授权，必须与官方资料交叉核验 |
| 开放研究/全球数据 | College Scorecard、IPEDS、Discover Uni、OpenAlex、ROR 等 | 院校画像、科研、成本、毕业率、国际对照和长期发展参考 | 不能直接用于中国高考投档概率 |
| 口碑线索 | 论坛、社交媒体、家长群、短视频 | 发现风险和异常线索 | 不得单独支撑结论 |

当前不假设存在统一、官方、稳定、公开可调用的全国招生计划或录取概率 API。正式填报仍以省级考试院、高校招生网和官方招生章程为准。

## 必须核验的信息

### 志愿填报

- 省份、年份、批次、志愿数量、志愿单位。
- 平行志愿或顺序志愿规则。
- 投档比例、同分排序、一轮投档逻辑。
- 院校专业组或专业+院校的调剂范围。
- 选科要求、体检要求、单科要求、语种要求。
- 招生计划数、专业组内专业、校区、学费。
- 近 2-3 年录取位次、最低分、平均分、专业录取差异。

### 院校与专业

- 办学性质：公办、民办、独立学院转设、中外合作。
- 主管部门和行业背景。
- 更名、合并、升格、转设、停招或校区变化。
- 一流专业、专业认证、硕博点、重点学科、行业平台。
- 就业质量、升学率、考研去向、系统内就业渠道。
- 转专业、辅修、第二学位、跨校选课政策。

### 教学质量与办学稳定性

- 学科评估结果、国家级/省级一流专业、专业认证。
- 本科教学质量报告、就业质量报告、升学去向和深造质量。
- 保研资格、保研率、考研率、考研通过率、研究生录取去向。
- 师资、实验平台、实训基地、校友网络和行业合作。
- 民办、独立学院、中外合作的转设、停招、收费和品牌风险。
- 新升本院校的办学底蕴、硕博点、学科建设和社会认可度。
- 联合培养、交换项目、国际合作、名校资源共享和双校园培养的官方证明。

教学质量证据不足时，不要仅因“录取稳”把选项放入保底。

## 升学跳板证据核验

当方案把某所学校作为保研、考研或继续深造跳板时，必须单独核验证据，不能只依赖口碑、家长群或“考研神校”标签。

优先信源：

1. 高校本科教学质量报告、毕业生就业质量报告。
2. 学校或学院的推免名单、公示、研究生录取去向公示。
3. 研究生院、本科生院、教务处、学院官网发布的推免资格、推免办法、竞赛科研和培养项目说明。
4. 招生章程、专业培养方案、硕博点和科研平台官方介绍。
5. 论坛、社交媒体、经验帖只能作为线索，必须交叉核验。

必须拆清口径：

| 数据 | 要核验什么 |
| --- | --- |
| 推免资格 | 学校是否有推免资格，相关专业或学院是否实际获得名额 |
| 推免率/保研率 | 分母是全校、学院、专业还是毕业生人数；年份是否连续稳定 |
| 推免去向质量 | 去向是否包括 985、211、双一流、强势学科高校或科研院所；是常态还是个例 |
| 考研通过率/录取率 | 是报名人数录取率、全体毕业生升学率，还是上线率、录取率混用 |
| 考研去向 | 录取学校层级、专业相关性、是否集中在本校或同层次学校 |
| 学风证据 | 课程要求、竞赛科研、导师资源、学习氛围、备考支持是否可见 |

输出纪律：

- “保研率高”不等于该学生能保研。必须写清需要保持专业排名、课程成绩、英语、科研竞赛和综合素质。
- “能去顶尖高校”只能作为上行情景。若只是少数个例，应标注为个例，不得写成普遍通道。
- 如果学生愿意读研，城市就业便利可以下调权重，但不能完全忽略升学失败后的就业退路。
- 如果官方数据缺失，只能写“待核验”，不能把第三方统计当确定结论。

### 民办、独立学院和名校背景核验

- 母体高校关系是否仍然存在，是否体现在官方文件中。
- 毕业证和学位证署名、办学性质、收费标准和校区。
- 转设、更名、停招、并入或举办方变化。
- 师资共享、课程共享、实验室共享、联合培养是否真实可用。
- 就业质量、考研去向和社会认可度是否能支撑高费用。

## 概率模型边界

本 Skill 默认不具备已回测校准的录取概率模型。用户要求精确概率、调档线预测或“几成能录”时，必须先判断是否具备以下条件：

- 该省该年完整官方招生计划或官方志愿系统导出。
- 近年同口径录取位次、最低分、平均分、专业组或专业录取数据。
- 专业组结构、招生人数、选科和限制条件的年度变化。
- 数据版本、来源等级、更新时间和冲突处理记录。
- 经过历史年份留出回测和校准的概率模型。

不满足以上条件时，只能输出：

- 冲刺、适冲、稳妥、强稳、保底、兜底等梯度。
- 概率区间或置信等级。
- 影响概率的正反因素。
- 数据缺口和待核验事项。

不得输出伪精确概率，例如“录取概率 87%”或“必上/必掉”。没有回测校准的模型结果必须标注为模拟假设。

当没有全国数据库和校准模型支撑时，必须直接告诉用户：本 Skill 并不真正掌握足够强的录取概率预测能力，方案更适合作为候选发现、风险审计和复核清单。正式填报前，建议把报告、候选表、专业组风险和待核验事项交给熟悉本省招生规则的高中老师、公益咨询志愿者、可信专业人士或官方志愿系统再次核对。

## 高杠杆假设审计

涉及特殊身份、行业系统、地理折价、末流高平台、学校合并、更名、升格、部委直属、保研资格、考研传统、体制内机会时，必须把判断拆开：

| 项目 | 核验问题 |
| --- | --- |
| 基本盘 | 不考虑上行故事，当前学校、专业组、城市、成本、学业压力是否可接受？ |
| 上行假设 | 更名、合并、行业扩张、政策红利、区域发展、学校升格是否有官方或权威证据？ |
| 兑现条件 | 需要几年、哪些公告、哪些政策或哪些学生行动才能兑现？ |
| 失败成本 | 如果假设不兑现，学生是否仍有考研、转专业、就业或城市迁移退路？ |
| 重新定价 | 近年录取位次是否已经上涨到当前考生不可达？ |

输出中必须区分：

- 机会雷达项：值得关注，但未必可达。
- 冲刺项：有概率但风险较高。
- 主推项：基本盘可接受，且证据和位次支持。
- 剔除项：方法论上有价值，但当前考生不适合或不可达。

不要把“候选池纳入”写成“建议填报”。候选池是扩大搜索空间，正式志愿必须经过规则和位次验证。

## 候选池覆盖与 ledger 核验

当任务涉及大量院校、院校专业组、专业或专业+院校单位时，必须建立候选池覆盖说明。

### 全量与高召回边界

- 只有拿到该省该年完整官方招生计划，并完成批次、科类/选科、志愿单位和限制条件标准化后，才可称为全量候选池。
- 若只使用用户意向学校、第三方工具、局部截图、往年资料或不完整表格，必须称为“基于当前数据源的高召回候选池”。
- 输出中不得暗示“已找到所有可能选项”，除非数据源范围确实支持。

### ledger 必查项

每轮候选处理都要核验：

- 如果用户提供 CSV、表格或已整理候选池，是否先使用 `scripts/ledger_tool.py` 做字段、状态、候选 ID、硬约束、最终表或覆盖缺口的机械校验。
- 候选池 ledger 是否记录来源类型、发现方式、进入理由和当前状态。
- 剔除/降级 ledger 是否记录剔除原因、证据、审查角色和复入条件。
- 待核验 ledger 是否记录影响结论、优先级、建议核验信源和截止节点。
- 最终组合 ledger 是否能追溯到候选池和风险审计。
- 新增候选是否先进入机会雷达或候选池，而不是直接进入最终志愿表。
- 被剔除候选是否有明确原因，且不会在后续轮次被遗忘或无理由复活。

脚本输出规则：

- `error` 表示结构或状态存在阻断问题，必须修正后再进入正式组合。
- `warn` 表示风险、缺口或过拟合线索，进入风险审计或待核验 ledger，不直接替代主 Agent 判断。
- 脚本过滤结果只代表显式硬约束命中，不代表学校质量或长期发展机会价值判断。

### 漏选审计

最终组合前必须检查是否遗漏：

- 区域外、非同源或偏远但层级、资源、升学或行业系统更强的候选。
- 特殊身份、主管部门、行业特色和系统认可候选。
- 同校其他专业组、同层次更安全专业组或专业+院校单位。
- 优质高职、职业本科、专升本、复读临界、艺体专项和定向等扩展画像路径。
- AI、交叉学科、区域国别、供应链、人口老龄化、绿色低碳、公共治理等宏观变量机会。

漏选审计发现的新候选必须记录来源和待核验事项。

### 过拟合审计

最终组合前必须检查是否过度迎合：

- 家长面子、本地便利、稳定幻觉或短期就业偏好。
- 学生一时情绪、热门城市、热门专业或网络叙事。
- 捡漏、升格、合并、更名、宏观趋势等上行故事。
- 第三方工具给出的单一概率、推荐标签或排名。
- 本省、本区域、熟悉城市、家庭照护半径或对区域外信息的不确定恐惧。
- 主 Agent 自己前一轮形成的偏见。

若发现过拟合，应补充替代候选、下调置信度或重新分配志愿名额。

### 第一性原理审查的证据边界

第一性原理审查只能用于重新打开候选空间，不能把未经核验的方向写成正式建议。

必须检查：

- 本质问题是否被拆成可验证的选择权，而不是抽象口号。
- 非同源对照组是否记录来源、发现方式、待核验项和复入条件。
- 地域惯性、概率崇拜、热门叙事或宏观故事是否已进入过拟合审计。
- 被误杀候选若缺少当年规则、招生计划、专业组或质量证据，只能进入机会雷达或待核验 ledger。
- 任何高杠杆候选仍需通过基本盘、上行假设、失败成本和学生承受力审计。

## 专业组调剂核验

对每个拟填专业组，必须核验：

- 组内全部专业名称、招生人数、学费、校区和选科要求。
- 组内专业是否同质，是否都指向同一行业或培养路径。
- 是否存在学生明确不可接受专业。
- 是否有身体、语种、单科、性别、政审、面试等限制。
- 专业分流、转专业、辅修和退出机制。
- 历史录取位次、专业冷热差异和大小年波动。

只有在“最差可接受结果”也可接受时，才建议服从调剂。

## 趋势与替代风险核验

涉及未来趋势时，必须同时检查机会和风险：

- 宏观假设：依赖 AI、产业、人口、地缘、区域战略或政策中的哪一个变量。
- 证据来源：官方政策、高校课程、就业质量报告、行业报告、可靠数据平台或口碑线索。
- 触发信号：政策支持、课程更新、行业招聘、学校平台、区域产业和学生能力出现正向证据。
- 反向信号：岗位收缩、课程空心化、录取重新定价、项目停滞、预算不足或学生执行失败。
- 兑现条件：学校课程、师资、平台、实习和学生能力能否支持该趋势。
- 失败成本：如果趋势不兑现，学生是否仍有考研、转专业、就业、证书或城市迁移退路。
- 时间尺度：短期就业热度和 10-20 年长期趋势必须分开说。

宏观趋势必须拆成保守、基准、上行三种情景；弱来源只能作为线索，不得单独支撑结论。

## 证据标签

输出表格中使用：

- 已刷新-当年官方：本轮已主动检索并查到当年官方资料，可作为当前最高优先级依据。
- 已核验-当年官方：已查到本年官方资料。
- 当年官方未发布：本轮已检索，但该项官方资料尚未发布，只能使用历史资料模拟。
- 疑似更新待核验：发现新消息或第三方提示，但尚未回到官方来源确认。
- 已核验-近年官方：近年官方资料可参考，但需更新。
- 商业 API：来自付费或商业数据服务，必须记录口径并与官方交叉验证。
- 开放研究数据：来自官方开放数据或研究机构数据，可用于院校画像和趋势参考。
- 辅助数据：来自可靠数据平台，需与官方交叉验证。
- 口碑线索：来自非官方讨论，仅作线索。
- 待核验假设：逻辑上可能成立，但尚无足够证据。
- 宏观情景假设：用于趋势推演，必须附触发信号和反向信号。

## 冲突信息处理

当信息冲突时：

1. 以当年官方资料为准。
2. 明确列出冲突来源。
3. 下调结论置信度。
4. 给出下一步核验动作。

当没有当年资料时：

1. 使用近年位次区间和招生计划趋势做模拟。
2. 固定加入免责声明。
3. 不给“确定录取”“必然捡漏”结论。

## 动态复核节点

出现以下情况必须刷新建议：

- 一分一段表发布。
- 省控线发布。
- 官方招生计划发布或变更。
- 院校专业组内容变化。
- 征集志愿或补录信息出现。
- 家庭预算、城市偏好、民办/中外合作容忍度变化。
- 学生本人改变专业或职业倾向。
- 真实位次数据证明早期冲刺选项过高。
- 特殊身份、主管部门、合并更名、保研资格、专业组或核心专业招生状态出现变化。
- 原本被低估的机会被市场重新定价，录取位次明显上移。
- AI、产业、人口、地缘、区域战略或政策变量出现新的触发信号或反向信号。
- 学校课程、师资、实验平台、实习资源或就业质量无法支撑原本的宏观逻辑。
- 新的官方招生计划或专业组信息证明候选池覆盖不完整。
- 漏选审计或过拟合审计发现当前组合存在结构性偏差。
