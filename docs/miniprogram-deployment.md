# 微信小程序部署指南

本文说明如何把 `miniprogram/` 目录发布为微信小程序线上版本。当前小程序是纯前端、本地存储版本，不连接后端，也不上传用户数据。

## 1. 发布前准备

1. 注册或登录微信公众平台小程序账号，并完成主体认证、名称、头像、简介和服务类目配置。
2. 在小程序后台获取 AppID，并把 `miniprogram/project.config.json` 中的 `appid` 从 `touristappid` 替换为真实 AppID。
3. 安装微信开发者工具，使用拥有该小程序开发权限的微信账号登录。
4. 根据微信平台要求准备隐私政策、用户协议、服务类目说明和审核演示材料。
5. 正式发布前再次确认本项目的非商用许可、免责声明和不得替代官方志愿系统的边界说明已经保留。

## 2. 本地导入与预览

1. 打开微信开发者工具，选择“导入项目”。
2. 项目目录选择仓库内的 `miniprogram/`。
3. AppID 填写真实小程序 AppID；仅本地体验时可使用游客模式。
4. 编译后检查三个页面：
   - `pages/index/index`：首页和入口说明。
   - `pages/intake/intake`：问诊资料填写、保存草稿、复制提示词。
   - `pages/disclaimer/disclaimer`：使用边界和正式填报前核验项。
5. 使用开发者工具“预览”生成二维码，用真机扫码测试页面跳转、本地保存和复制提示词。

## 3. 上传代码

1. 在微信开发者工具中确认当前项目没有编译错误。
2. 点击“上传”，填写版本号和项目备注，例如：
   - 版本号：`0.1.0`
   - 项目备注：`新增 Lever-GaoKao 问诊与边界说明小程序壳`
3. 上传后登录微信公众平台，在版本管理中确认出现对应的开发版本。

## 4. 提交审核

1. 在微信公众平台进入小程序版本管理，选择刚上传的开发版本并提交审核。
2. 审核资料建议填写：
   - 功能页面：`pages/index/index`
   - 功能说明：高考志愿填报资料收集、风险提示与提示词复制工具。
   - 测试说明：说明当前版本不需要登录、不连接后端、不上传用户数据；问诊草稿只保存在用户本机。
3. 如平台要求填写隐私处理，请按真实功能声明：当前版本不收集、不上传、不共享个人信息；本地草稿仅存于用户设备。
4. 如后续接入后端、登录、表单提交、客服或数据分析，需要重新补齐隐私协议、服务器域名、接口安全和数据删除机制。

## 5. 发布上线

1. 审核通过后，在微信公众平台点击“发布”。
2. 发布后使用不同微信账号搜索小程序，检查首页、问诊页、边界页是否可正常访问。
3. 建议保留一个体验版本给维护者回归测试，线上版本只发布已经通过真机检查的代码。

## 6. 常见问题

### 为什么现在没有后端？

当前项目优先降低部署门槛，避免处理考生隐私数据。问诊资料只保存在本机，并通过复制提示词交给用户自己选择的 Agent、老师或公益咨询志愿者。

### 可以直接收集用户表单吗？

不建议在没有隐私政策、数据安全方案、删除机制和合规主体前收集。高考志愿资料包含较强个人和家庭信息，正式上线收集前应完成最小化采集、加密传输、权限控制、留存期限和删除入口设计。

### 可以接入 AI 接口自动分析吗？

可以作为后续版本，但需要先补齐后端服务、模型调用成本控制、内容安全、官方资料核验、隐私合规和“非录取保证”的强提醒。不要在前端小程序内暴露任何模型 API Key。
